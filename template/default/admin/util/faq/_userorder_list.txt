受注台帳
+ 受注台帳について
ユーザが購入した商品について、受注情報の検索や管理を行うことができます。

+ 注文番号とは何ですか？
受注情報毎に指定されているIDのことです。商品購入時にシステム側でユニークな整数が付与されます。管理画面からの変更はできません。

+ 注文者とは何ですか？
商品を注文したユーザのことです。注文者名を押下することでユーザ情報の編集を行うことができます。

+ 注文ステータスの未決済、決済済み、注文済み、返品済み、キャンセルはそれぞれどのような状態ですか？
以下の通りです。
未決済　　　…　決済が完了していない状態
決済済み　　…　決済が完了し商品代金が支払われた状態
注文済み　　…　メーカーに商品を発注した状態
返品済み　…　購入者が商品を返品した状態
キャンセル　…　購入者が商品をキャンセルした状態

+ 商品名とは何ですか？
ユーザが購入した商品の名称のことです。商品名を押下することで商品情報の編集を行うことができます。

+ タイプとは何ですか？
商品に複数のタイプ（色・サイズ等）が設定されている場合、購入時に選択したタイプが表示されます。

+ 受注情報を編集するには？
対象となる受注情報の注文番号を押下します。

+ お届け先を変更するには？
商品お届け先が「指定住所」となっている受注情報については、受注詳細情報編集画面よりお届け先氏名・住所・電話番号の変更が可能です。「注文者住所」となっている場合は顧客情報に設定された住所がお届け先住所となります。

+ お支払方法を変更することはできますか？
できません。

+ ポイント利用とは何ですか？
商品の購入時に「ポイントを使用する」が選択されていた場合、ユーザが設定した利用ポイントが表示されます。設定されたポイント分だけ商品合計金額（税抜）から割引されます。

+ 獲得ポイントとは何ですか？
商品に設定された「ポイント」が獲得ポイントとしてユーザのポイントに付加されます。付加すべきポイントの確定は、ユーザ側「この内容で注文する」ボタンを押下した時点で行われます。

+ 獲得ポイントが実際に付与されるタイミングは？
実際のポイント付与は、管理側受注詳細編集の商品発送ステータスを「発送済み」に更新し、「ほかに発送すべき商品がなくなった時点」でポイントの付与を行っております。

+ 注文商品の数量を変更するには？
受注詳細情報編集画面より対象となる商品の数量を変更することができます。数量を変更すると注文金額が再計算されて表示されます。

+ 同時発送グループとは何ですか？。
同時発送が可能な商品を同一伝票で管理するためのグループ分けとなります。

+ 商品発送ステータスの未発送、発送済みはそれぞれどのような状態ですか？
以下のとおりです。
未発送　　…　商品を発送していない状態
発送済み　…　商品の発送が完了した状態
